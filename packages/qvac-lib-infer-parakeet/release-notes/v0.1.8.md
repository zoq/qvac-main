# QVAC Transcription Parakeet Addon v0.1.8 Release Notes

This release fixes a race condition where the native addon changed the process-wide working directory during model loading, causing sporadic audio file resolution failures when using relative paths.

## Bug Fixes

### Remove process-wide chdir from ONNX session loading

The `ParakeetModel::load()` method used `std::filesystem::current_path()` to change the process CWD so ONNX Runtime could locate external `.onnx.data` files alongside the encoder model. Because `load()` runs on a background thread, this changed the CWD for the entire process — including the main JS thread — creating a race condition that caused `AUDIO_FILE_NOT_FOUND` errors when users passed relative audio file paths (e.g. `./audio/sample.wav`).

ONNX Runtime resolves external data files relative to the model file's directory when given absolute paths, so the `chdir` was unnecessary. For the named-paths branch, the staging directory already places both `.onnx` and `.onnx.data` symlinks side by side.

**Affected code paths:**
- Named-paths branch (individual file paths via registry) — `chdir` to staging directory removed
- Directory-path branch (all files in one directory) — `chdir` to model directory removed

## Backward Compatibility

No API or behavioral changes. All existing callers continue to work without modification.
