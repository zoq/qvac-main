# QVAC OCR Addon v0.1.3 Release Notes

This release introduces automated GitHub releases with mandatory release notes validation, ensuring every version bump is accompanied by proper documentation.

## Features

### Automated GitHub Releases

GitHub releases are now automatically created when a version bump is detected on merge to main. The release workflow reads from the corresponding `release-notes/vX.Y.Z.md` file and creates a tagged release with the documentation included.

### Release Notes Validation

A new PR check ensures that any pull request containing a version bump also includes the matching release notes file. This prevents releases from being published without proper documentation.

### Dual Registry Publishing

The prebuild workflow now supports a `both` option for `publish_target`, allowing simultaneous publishing to both npm and GitHub Package Registry in a single workflow run.

## Infrastructure

### Version Bump Detection Action

A new reusable GitHub Action (`.github/actions/detect-version-bump`) compares the `package.json` version between commits to determine if a version bump occurred. This action is used by both the release workflow and the PR validation check.

### AI-Assisted Release Notes Commands

New slash commands have been added for Cursor and Claude Code to help generate PR descriptions (`/prepare_pr`) and release notes (`/prepare_release_notes`), streamlining the release documentation process.
