# QVAC Transcription Whispercpp Addon v0.4.2 Release Notes

This release aligns the addon's logger type with the shared `LoggerInterface` from `@qvac/logging`, enabling the SDK to pass its logger directly without type casting.

## Changes

### Align logger type to shared LoggerInterface

The `logger` field in `TranscriptionWhispercppArgs` now uses `LoggerInterface` from `@qvac/logging` instead of a package-specific logger type. This ensures type compatibility with the SDK's logging infrastructure and removes the need for `as unknown as` casts when constructing addon arguments.

## Backward Compatibility

No runtime behavioral changes. The `LoggerInterface` type is structurally compatible with the previously accepted logger shape, so all existing callers continue to work without modification.
