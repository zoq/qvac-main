## Overview

[Completion](/v0.7.0/sdk/ai-tasks/completion) supports multimodal prompts. Multimodal lets you attach media files to inputs for [`completion()`](/v0.7.0/sdk/api/completion). You can include multiple attachments in the same request (e.g., to compare two images).

Compared to text-only completion inference, the key differences are:
- You must load a multimodal-capable LLM **and** its matching `projectionModelSrc` via [`loadModel()`](/v0.7.0/sdk/api/loadModel).
- Your `history` messages can include `attachments: [{ path: "/path/to/image.jpg" }]` (the file must exist on disk).
- Aside from attachments, you still call `completion({ modelId, history, stream })` the same way and consume the same streaming output.

## Functions

Use the following sequence of function calls:
1. [`loadModel()`](/v0.7.0/sdk/api/loadModel)
2. [`completion()`](/v0.7.0/sdk/api/completion)
3. [`unloadModel()`](/v0.7.0/sdk/api/unloadModel)

For how to use each function, see [SDK — API reference](/v0.7.0/sdk/api/).

## Models

You should load two models:
- a `llama.cpp`-compatible multimodal-capable LLM. Model file format: `*.gguf`; and
- a matching projection model (`mmproj-*.gguf`). Model file format: `*.gguf`.

Recommended pairs:
- SmolVLM2 + mmproj-*
- Qwen2.5-Omni + mmproj-* (or Qwen3-VL + mmproj-*)

For models available as constants, see [SDK — Models](/v0.7.0/sdk#models).
